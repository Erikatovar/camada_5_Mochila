a. ¿Quién inventó el sistema de control de versión Git y por qué?
    Git fue creado por Linus Benedict Torvalds, un ingeniero de software inlandés-estadounidense quien es conocido por haber iniciado y mantenido el desarrollo de «kernel» (núcleo) de Linux.
    Él creó este software porque el único que había disponible en la época se llamaba «BitKeeper» y no era un software libre; él buscaba un software que fuera libre y que tuviera buen desempeño, al no encontrar esas características en los software libres que había, lo empezó a crear él mismo y tuvo un crecimiento enorme, siento tan bueno como los software bajo licencia.
b. ¿A quién pertenece actualmente Github y por qué?
    GitHub pertenece a Microsoft desde el 2018, el objetivo de la compra de la plataforma fue porque GitHub era (desde el 2018) la plataforma lider en desarrollo de software; en sí, lo que dice Microsoft es que su interés era apoyar a las comunidades puesto que «Los desarrolladores son los constructores de esta nueva era» y también porque piensa que el destino para que los desarrolladores aprendan, compartan y trabajen juntos para crear software». Yendo a la parte monetaria, la adquisición de GitHub sirvió para potenciar los servicios digitales y la tecnología «cloud computing».
c. ¿Hay otra forma que no sea la terminal para trabajar con Github?
    Sí, se puede crear un repositorio sin necesidad de comandos desde www.github.com siempre y cuando se marque la opción de README.md; otro método es mediante la app GitHub Desktop, que es una manera de crear y administrar un repositorio sin utilizar la línea de comandos en la terminal, incluso se puede colocar el editor favorito en las opciones del programa.
